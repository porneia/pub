gapz.tuxfamily.org, 30 juin 2006, {{ page.author }} — Vous aurez sans doute déja remarqué que lorsque vous vous connecter sur IRC vous avez après votre pseudo quelque chose qui ressemble à ceux-ci par exemple : 

(n=pseudo@Adunkerk-231-4-88-153.w73-221.abo.wanadoo.fr)

Chez certain FAI vous avez directement votre IP qui s'affiche au lieu de "Adunkerk-231-4-88-153.w73-221.abo.wanadoo.fr" dans notre exemple. Si vous voulez éviter toutes sortes d'attaque vous en avez les moyens avec le lociel TOR. Si votre distribution est basé sur "des .deb" et que tor ce trouve dans vos dépôts, alors : 

$ sudo apt-get install tor

Ensuite il faut "tor-iffier" votre client IRC, par exemple si vous utilisez xchat, aller dans les préférences, réglage relatif au réseaux, puis comme hostname vous mettez : 127.0.0.1 et comme port : 9050, et enfin de type Socks5, laisser les cases non remplis vide. Si vous utiliser un autre client, vous pouvez utiliser le guide de tor-iffication. Par contre, sur certain réseaux vous aurez des problèmes de bannissement d'ip etc... Donc il se peut que la connection soit plus longue. Malgrès tout, si vous êtes un utilisateur du réseaux freenode, au lieu de ce connecté sur irc.freenode.org connecter vous plutôt : 6ua4nhltph56henu.onion Port 6667.

Si vous n'utilisez pas ce réseaux que vous trouvez les connexions trop longue, mettez ces quelques lignes au début de votre /etc/tor/torrc : 

ClientOnly 1
TrackHostExitsExpire 30
MaxCircuitDirtiness 30
TrackHostExitsExpire 120

Adapter ces lignes en fonction ce que vous désirez ;-)
